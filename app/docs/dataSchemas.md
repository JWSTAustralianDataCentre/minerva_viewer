# RISKS
- Flux units are '10**1 nJy' (=10 nJy per unit), a non-standard astropy unit string - the serving layer must multiply by 10 to convert to nJy. Applies to all f_/e_/faper/flux_ratio columns in SUPER_CATALOG.
- In eazy.data.fits the CHI2 (607,294126) and COEFFS (15,294126) arrays are stored transposed relative to the h5 (nz/ntemp on axis 0, object on axis 1). Single-object slice = arr[:, i] in data.fits but arr[i, :] in the h5. Do not assume same axis order.
- p(z) must be COMPUTED, not read: no stored p(z) or zgrid dataset in the h5. p(z) ~ exp(-(chi2_fit[i]-min)/2) over the 607-pt log zgrid; zgrid is only explicit in data.fits HDU 'ZGRID' or regenerable from Z_MIN/Z_MAX/Z_STEP/Z_STEP_TYPE=1(log) attrs. Prior is NOT applied (APPLY_PRIOR=0).
- SFHZ templates are redshift-dependent: each 'corr_sfhz_13_bin*' template has flux shape (14,9300) with a z-array of 14 values, so reconstructing an SED at zbest requires interpolating along the template's own z axis - not a single static spectrum. Use eazy.hdf5.templates_from_hdf5 rather than reading raw arrays.
- coeffs_best has 15 columns; template ordering = order of datasets in templates/ group and axis-1 of tempfilt (13 sfhz + fsps_4590 + j0647agn+torus). Verify ordering matches before combining coeffs with templates.
- SUPER_CATALOG lives OUTSIDE the EAzY dir; the h5/zout/data.fits are under EAzY/<TEMPLATESET>/SUPER/ZPiter/. A parallel D020 aperture variant exists (D020_CATALOG, likely different row count) - do not confuse D020 files with SUPER files when globbing.
- csv 'grade' has 38015 NaN of 80367 rows (~47% ungraded); any quality filter on grade must decide NaN handling explicitly.
- chi2_fit is the largest per-object-addressable array (294126 x 607 x f32 ~714MB fully); always slice by row via h5py, never load whole. Cross-object queries (p(z) for many objects) will be I/O bound.
- z_spec exists in SUPER_CATALOG and zout/h5 but is likely mostly filler (-1/NaN); real spectroscopic redshifts should come from the emission-lines csv (z_best) matched by ra/dec, not from the photometric catalog's z_spec column.
- COSMOS and old-UDS catalogs differ in filter set (UDS has f775w, COSMOS has f070w/f275wu/f336wu/f475w) and flag columns, so a unified schema across fields must treat the band set as field-dependent, not fixed.
