# SUMMARY
index.html is not a hand-authored page but a 557KB 'Classical' ds-bundle: a JSON manifest of gzip+base64 resources (React 18.3.1, ReactDOM, dc-runtime framework JS, and 12 embedded woff2 fonts for Cormorant Garamond + Lora) plus a JSON-escaped HTML template. The actual app is ~64KB: CSS design tokens in a <helmet><style> block, {{ }}-bound markup in <x-dc>, and a single 257-line JavaScript class `Component extends DCLogic` (a React-style component with setState + lifecycle) holding the entire mock data layer. All mock generators are methods on that class. makeObjects() returns 100 seeded-random object records (fields: i, dja[id], mid[minerva id], ra, dec, sep, zs, zp, grade, plus synthesis-only ztrue/kind/slope/lstr/f0/mag/sn). spec(o) returns memoized {w,f,e} plain arrays (380 pts, μm/μJy) via this.specCache — the cleanest swap seam. drawAll() side-effect-draws a FAKE 2D spectrum (1D smeared on a 1000x86 canvas) and a synthetic cutout (180x180 canvas). renderVals() returns an ~80-key flat view-model, synthesizing SED (sedPts as pixel coords + sedTemplPath) and P(z)/chi2 (path strings, no grid array) INLINE — these are emitted directly as SVG pixel geometry, so they are not drop-in array swaps. Objects flow objs -> filtered()(threshold |dz|/(1+zs)>thresh, grade filters, search, sort) -> rows/scatter -> state.selId(o.dja) -> cur() -> inspector. Decisions persist to localStorage ('djaqc-decisions','djaqc-by'); CSV export matches the HANDOFF 14-column spec. Keyboard 3/2/1/f/0/n/p/m/r wired in onKey. The code is fully synchronous with no loading states, but the React setState substrate means an async data provider can be injected — the specCache is the natural seam — provided loading/error states are added and the SED/pz inline synthesis in renderVals is rewired to thread real physical arrays through the existing pixel transforms. Extracted, decoded copies of template.html, app.js, and the runtime are in the scratchpad proto/ directory for the next engineer.

# RISKS
- HANDOFF field names are loose vs reality: the object id key is `o.dja` (string), NOT `o.id`; MINERVA id is `o.mid`. Any real-data mapping must use these exact keys or the whole render/persist/CSV chain breaks (decisions map, specCache, select/step all key on o.dja).
- SED photometry (sedPts) and P(z)/chi2 are NOT exposed as physical arrays — they are computed inline in renderVals() and emitted directly as SVG pixel paths/coords. Swapping them is not a drop-in array replacement; it requires editing renderVals to read real arrays and pass them through the existing sxx/syy/pzx pixel transforms. This is the biggest hidden coupling.
- 2D spectrum is faked from the 1D flux (vertical Gaussian smear on canvas), not a real 2D array. Real FITS 2D SCI needs new draw code in drawAll and must share the 1D wavelength WCS to preserve the linked zoom (state.wmin/wmax drive both).
- Fully synchronous rendering: spec()/drawAll()/renderVals assume instant data. Injecting async (S3 FITS fetch, eazy JSON, grizli cutout) requires ADDING loading/error states everywhere; none exist. componentDidUpdate calls drawAll() on every render, so a naive async spec() returning a Promise would draw nothing/throw.
- Hardcoded canvas/SVG pixel sizes and transforms throughout: 2D 1000x86 (draw area x=48, width 944), cutout 180x180, scatter 298x252 with sx/sy transforms, 1D 944x338, SED sxx(0.45-5.6μm log)/syy, pz pzx(0..zmax*394). Real data with different wavelength ranges/aspect must respect these fixed geometries or retune the transforms.
- CSV export iterates ALL this.objs and hardcodes filename 'dja_minerva_qc_cosmos.csv' (field='cosmos' baked in) — the field switcher (COSMOS/UDS/EGS) changes state.field but the export filename does not follow it.
- The whole app is inside a proprietary 'Classical' ds-bundle (dc-runtime, template DSL with {{ }} bindings and <x-dc>). It is NOT plain HTML — an engineer cannot just edit index.html; they must edit the source that regenerates the bundle, or carefully patch the JSON-escaped template string. The runtime expects window.React/ReactDOM globals loaded first.
- Zoom/pan clamps are hardcoded to the mock wavelength window (wmin>=0.55, wmax<=5.35, span limits 0.035..4.75) in the wheel handler and onSpecDown — real spectra outside 0.6-5.3μm would be clipped.
- specCache is unbounded and keyed by o.dja with no eviction — fine for 100 mock objects but must become an LRU (HANDOFF mentions zgui_panel.py LRU + prefetch) for real catalogs of thousands.
- Marker/line-confusion and outlier populations are synthesized via o.kind/ztrue/slope/lstr/f0 which have no real-data source; removing them is fine but any code path reading o.kind (e.g. P(z) secondary peak, cutout blob offset, sn floor for 'match') must be updated when real data replaces the synthesis.
