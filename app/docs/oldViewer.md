# SUMMARY
Two-part Python astronomy tool for the JWST MINERVA survey (NIRCam medium-band imaging over Hubble legacy fields, MINERVA-UDS field). (1) minerva_viewer.py is a one-shot batch analysis script: loads photometric + SPS + multi-template EAzY photo-z catalogs, cross-matches against the public DJA/MSAexp NIRSpec spec-z catalog via astropy SkyCoord, and produces a photo-z-vs-spec-z performance plot (matplotlib) with NMAD/outlier statistics. (2) zgui_panel.py is a Panel (HoloViz) single-page web app for interactively browsing individual galaxies (cutout image, EAzY SED fit, NIRSpec spectrum), backed by a thread-safe in-memory LRU cache and a background ThreadPoolExecutor prefetcher, using pn.cache for one-time global data loads. config.py is a 4-tier (env var > local config.ini > ~/.minerva/config.ini > hardcoded defaults) singleton path-resolution module shared by both scripts. setup_paths.py is an interactive/CLI wizard that writes config.ini. All catalog filenames/subpaths are hardcoded to the MINERVA-UDS field's specific naming convention (v1.0, n2.2, m2.0, LW_Kf444w, SUPER). No tests, no CI, no requirements pinning beyond requirements.txt (not read in detail but referenced).

- LRU CACHE IMPLEMENTATION (zgui_panel.py, class LRUCache): backed by collections.OrderedDict + threading.Lock (fully serialized get/put/contains under one lock — no per-key locking). get(key): if present, move_to_end (marks MRU), increments self.hits, returns value; else increments self.misses, returns None. put(key,value): if key exists, move_to_end and overwrite; else insert, then if len(cache)>maxsize, popitem(last=False) evicts the LRU (oldest) entry. stats() returns a formatted string 'Cache: {n}/{maxsize} items, {hit%}% hit rate'. THREE global singleton caches created at module load (persist across Panel sessions/browser tabs since they're module-level, not per-session): CUTOUT_CACHE=LRUCache(maxsize=300) (~1-5MB/item, ~1.5GB max per comment), SED_CACHE=LRUCache(maxsize=300), SPECTRUM_CACHE=LRUCache(maxsize=300). Cache keys: cutout key = f'cutout_{gal_id}_{size}_{scale}' (so changing size/scale busts cache), sed key = f'sed_{gal_id}', spectrum key = f'spectrum_{gal_id}'.
- BACKGROUND PREFETCH THREAD MODEL: single module-level PREFETCH_EXECUTOR = ThreadPoolExecutor(max_workers=4), shared across all sessions/instances. Per-app-instance state: self._prefetch_lock (threading.Lock), self._prefetching (a set of galaxy IDs currently being prefetched, guarded by the lock, used to prevent duplicate concurrent prefetch of the same id), self._display_token (a float timestamp via time.time(), bumped by _next_display_token() on every navigation event) used to discard stale async results — when a background fetch completes, the callback checks token != self._display_token and no-ops if the user has since navigated away. _async_update_pane(pane, fetch_fn, token): sets pane.loading=True, submits a task to PREFETCH_EXECUTOR that calls fetch_fn() in the thread, then uses _safe_ui_update() to marshal the pane.object assignment back onto Panel's document loop via doc.add_next_tick_callback (falls back to calling directly if pn.state.curdoc is falsy, e.g. no active session). _schedule_prefetch(): on every navigation, computes offsets [1,2,3,-1] from current index (prefetch next 3 + previous 1), submits _prefetch_galaxy(gal_id) to the executor for each not already cache-hit for the current size/scale. _prefetch_galaxy(gal_id) fetches cutout+SED+spectrum sequentially in one thread task, guarded by the _prefetching set to dedupe. # RISKS
- Two scripts define 'detected' sources differently (minerva_viewer.py adds an n_bands_mb>4 filter that zgui_panel.py's load_photometric_catalog() omits) — a rewrite must decide/reconcile a single source of truth for this filter.
